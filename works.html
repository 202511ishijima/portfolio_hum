作品概要 必須
どんなサービス・サイトか、ターゲットユーザーは誰か

ハムスターカフェ、ハムスターのお迎え案内、飼育用品のEC、会員制度を組み合わせた架空サービスのWebサイトです。

主なターゲットは、ハムスターに興味を持つ20代女性と、小学生低学年の子どもを持つ保護者です。

演習1では静的サイトとして企画・設計・UI制作を行い、演習2では既存デザインを活かしたまま Spring Boot を用いたお問い合わせ管理・会員管理機能を追加しました。

使用技術 必須

フロントエンド：HTML / CSS / JavaScript / JSON / localStorage

バックエンド：Java / Spring Boot / MyBatis / Spring Security / JUnit

データベース：H2 Database（開発用） / MySQL想定でDB設計書を作成

その他：Git / GitHub / GitHub Pages / Figma / VS Code

担当範囲 必須
企画・デザイン・フロントエンド・バックエンドなど

企画立案、ターゲット設定、サイトマップ作成、ワイヤーフレーム作成、画面デザイン、フロントエンド実装、バックエンド実装、DB設計、テスト、README整備まで一通り担当しました。

特に、演習1で作成した静的サイトのデザインを大きく崩さずに、演習2で管理画面やお問い合わせ機能を追加する点を意識して進めました。

工夫した点・アピールポイント 必須

フロントエンドでは、やわらかい世界観を維持しながら、カフェ、ハムスター一覧、商品一覧、会員制度など情報量の多いページを整理し、初見でも回遊しやすい構成にしました。

JavaScript では JSON データを用いた一覧表示、localStorage を使った疑似カート、購入情報の保存、ログイン状態の保持など、利用者が動きを体験できる仕組みを実装しました。

バックエンドでは Spring Boot + MyBatis でお問い合わせ管理と会員管理を構築し、管理画面から問い合わせ対応、会員情報編集、停止・ポイント調整・削除ができるようにしました。

設計資料も `docs/` にまとめ、企画から設計、実装、テストまでの流れを一つのリポジトリ内で確認できるようにしています。
