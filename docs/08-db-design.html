1 設計方針

対象範囲 必須

本DB設計は、課題2で追加する「お問い合わせフォーム」「管理画面での問い合わせ一覧表示」を中心に構成する。

今後の拡張として「作品管理CRUD」を追加できるよう、works テーブルと関連設計も含める。

DB は MySQL を想定し、バックエンドは Spring Boot + MyBatis で実装する前提とする。

2 ER図

ER図 必須
admins
 └ admin_roles

inquiries

works
 └ work_tags

tags
 └ work_tags
ER図の説明 必須

inquiries は公開側から送信された問い合わせ情報を保存する中心テーブルである。

admins は管理画面ログイン用の管理者テーブルであり、admin_roles によって権限種別を管理する。

works は将来の作品管理CRUD用テーブルであり、tags / work_tags によりタグ付けできる構成とする。

4 MyBatis設計

Mapper設計方針 必須

`InquiryMapper` では、問い合わせ登録、一覧取得、詳細取得、対応状況更新を担当する。

`AdminMapper` では、管理者ログインに必要な管理者情報と権限情報の取得を担当する。

`WorkMapper` では、将来の作品一覧、登録、更新、削除のCRUD処理を担当する。

想定Mapperメソッド 必須
InquiryMapper
- insertInquiry(Inquiry inquiry)
- findAllInquiries()
- findInquiryById(Long id)
- updateInquiryStatus(Long id, String status)

AdminMapper
- findAdminByLoginId(String loginId)
- findRolesByAdminId(Long adminId)

WorkMapper
- findAllWorks()
- findWorkById(Long id)
- insertWork(Work work)
- updateWork(Work work)
- deleteWork(Long id)

5 CREATE TABLE / 初期データ

CREATE TABLE方針 必須

MySQL 8 系を想定し、主キーは BIGINT AUTO_INCREMENT を用いる。日時は DATETIME で管理する。

問い合わせステータスは `NEW`, `IN_PROGRESS`, `COMPLETED` を文字列で保持する。
