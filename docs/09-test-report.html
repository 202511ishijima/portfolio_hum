1 テスト方針

テスト対象 必須

本テスト報告書は、課題2で実装するお問い合わせ登録機能、管理者ログイン機能、問い合わせ一覧 / 詳細表示機能を対象とする。

あわせて、将来的に追加予定の作品管理CRUDについても拡張しやすいよう、テスト観点を先に定義する。

テスト観点 必須

1. 正常系: 正しい入力で処理が成功すること

2. 異常系: 必須未入力、形式不正、権限不足などで適切にエラーとなること

3. セキュリティ: 未認証アクセスを遮断できること

4. データ整合性: 登録、更新内容がDBへ正しく反映されること

3 API単体テスト項目

API単体テスト観点 必須

`POST /api/inquiries` では、正常登録、必須チェック、メール形式チェック、文字数上限チェックを行う。

`GET /api/admin/inquiries` では、認証あり / なし、データ件数0件 / 複数件を確認する。

`GET /api/admin/inquiries/{id}` では、正常取得、存在しないID、未認証アクセスを確認する。

`PATCH /api/admin/inquiries/{id}/status` では、正常更新、存在しないID、不正なステータス値を確認する。
