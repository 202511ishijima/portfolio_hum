1 システム概要

前提 必須

本仕様書は、演習1で制作したポートフォリオサイトを前提に、バックエンドを追加実装するためのたたき台である。

既存のHTML/CSSデザインはできるだけ維持し、Spring Boot + MyBatis を用いて、お問い合わせ機能と管理画面機能を追加する。

将来的には作品管理CRUDを追加できるよう、API設計とDB設計は拡張しやすい構成とする。

今回の対象機能 必須

1. 公開側のお問い合わせフォーム送信

2. 管理者ログイン

3. 管理画面でのお問い合わせ一覧表示

4. お問い合わせ詳細表示

5. 将来拡張として作品管理CRUDを追加できる構造

3 画面仕様

公開側: お問い合わせページ 必須

画面URL: `/pages/contact.html`

入力項目: 氏名、メールアドレス、件名、問い合わせ本文

送信ボタン押下時にAPIへPOSTし、成功時は完了メッセージまたは完了ページへ遷移する。

入力エラー時は各項目の下、またはフォーム上部にエラーメッセージを表示する。

管理側: ログイン画面 必須

画面URL: `/admin/login`

入力項目: ログインID、パスワード

認証成功時は問い合わせ一覧画面へ遷移し、失敗時はログイン画面にエラーメッセージを表示する。

管理側: 問い合わせ一覧画面 必須

画面URL: `/admin/inquiries`

表示項目: 受付日時、氏名、メールアドレス、件名、対応状況

機能: 詳細画面への遷移、対応状況の絞り込み、将来的なページング対応

管理側: 問い合わせ詳細画面 必須

画面URL: `/admin/inquiries/{id}`

表示項目: 氏名、メールアドレス、件名、問い合わせ本文、受付日時、対応状況

機能: 対応状況更新、一覧へ戻る

5 バリデーション・セキュリティ

バリデーション要件 必須

氏名: 必須、100文字以内

メールアドレス: 必須、メール形式、255文字以内

件名: 必須、150文字以内

本文: 必須、1000文字以内

管理者ログインID / パスワード: 必須

セキュリティ要件 必須

Spring Security を用いて `/admin/**` と `/api/admin/**` を認証必須とする。

パスワードは平文保存せず、BCrypt などでハッシュ化する。

CSRF対策、認可制御、入力値検証を行い、不正アクセスや不正入力を防止する。
