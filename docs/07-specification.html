1 システム概要

システムの目的 必須

本システムは、架空サービス「はむのくらし」の紹介、ハムスター一覧表示、関連商品の疑似EC、会員制度案内を行う静的 Web サイトである。

利用者がカフェ体験からハムスターへの興味を持ち、生体情報を確認し、飼育用品を購入し、会員制度によって継続的に利用したくなる流れを作ることを目的とする。

システム構成 必須

本システムは HTML / CSS / JavaScript のみで構成された静的サイトである。

商品情報、ハムスター情報、ニュース、景品情報などは JSON ファイルから取得し、画面表示時に JavaScript で描画する。

カート情報、会員状態、直近注文金額は localStorage に保存し、簡易的な疑似EC機能として実装する。

主な利用フロー 必須

利用者はトップページからカフェ紹介やハムスター一覧へ進み、興味を持ったあとに商品ページでスターターセットや飼育用品を選ぶ。

商品をカートへ追加したあとは、カート、購入手続き、購入完了へ進み、購入後は会員制度やマイページを確認する流れを想定している。

4 機能詳細

商品一覧・商品詳細

入力 / 出力 / 動作 必須

入力: 商品カテゴリ選択、カート追加ボタン、数量の増減ボタン

出力: 商品名、説明、価格、画像、カテゴリ、現在の数量表示

動作: JSON から商品データを取得し、一覧・詳細を描画する。カート追加後は localStorage に保存し、画面上の数量とヘッダーのカート件数を更新する。

購入手続き

入力 / 出力 / 動作 必須

入力: 名前、郵便番号、住所、電話番号、支払い方法

出力: 注文サマリー、小計、送料、合計金額

動作: 郵便番号から住所検索を行い、電話番号は数字のみ受け付ける。送信後は購入完了ページへ遷移する。

ログイン・会員登録・マイページ

入力 / 出力 / 動作 必須

入力: 氏名、メールアドレス、パスワード

出力: 会員名、現在ポイント、景品ページへの導線

動作: localStorage に会員情報を保存し、ログイン状態を保持する。ログイン中はヘッダーにアカウント名を表示し、マイページからログアウトできる。

ハムスター一覧・詳細

入力 / 出力 / 動作 必須

入力: 種類タグ、詳細リンク

出力: 種類名、性格、飼いやすさ、価格、説明、画像

動作: JSON からハムスターデータを取得し、一覧と詳細を表示する。種類タグから詳細ページへ遷移できる。

5 非機能要件

対応ブラウザ 必須

Google Chrome、Microsoft Edge を主な確認対象とする。実装はモダンブラウザ環境での動作を前提とする。

デザイン要件 必須

スマホファーストで設計し、ベージュ系ナチュラル上品を基調とする。ブランドトーンは「上品 60% + 可愛い 40%」とし、SNS へ投稿したくなる雰囲気を意識する。

運用上の前提 必須

本システムはローカルサーバー上で動作確認を行う。JSON 読み込みの都合上、file:// 直開きではなく http://127.0.0.1:3000/ などのサーバー経由で閲覧することを前提とする。
