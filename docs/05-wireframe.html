1 ワイヤーフレーム作成方針

設計方針 必須

スマホファーストを前提に、縦スクロールで情報を順番に理解できる構成とする。

トップページでは「癒し体験 → 興味喚起 → お迎え → 商品購入 → 会員制度」の導線を自然につなげる。

カフェ、生体、お迎え、商品、会員制度の各ページは、目的ごとに情報を整理し、迷わず次の行動へ進めるレイアウトにする。

本資料では提出用として、画面構成が分かるテキスト型ワイヤーフレームを記載し、後から Figma 画像へ差し替えやすい内容で整理する。

ページ後半にはお知らせや SNS 導線を配置し、定期的な再訪や拡散につなげる。

5 スマートフォン対応

スマートフォン対応の考え方 任意

ヘッダーはハンバーガーメニューを採用し、主要導線である「カフェ」「お迎え」「商品」「会員制度」に1タップで移動できる構成とする。

カード一覧は1カラムを基本とし、商品や生体はスクロールで比較しやすいようにする。

CTA ボタンは親指で押しやすい大きさで下方向に配置し、各セクションの終わりで次の行動が分かるように設計する。

文章量の多いページでは、見出し・カード・余白を使って情報を区切り、スマホでも読み疲れしにくい画面構成にする。
